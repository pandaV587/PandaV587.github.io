Linux系统jdk版本替换

发表于 2018-05-07 | 更新于 2018-05-14 | 分类于 环境部署 | 评论数：
本文字数： 688 | 阅读时长 ≈ 1 分钟

　　在环境部署的过程中，经常遇到jdk版本不匹配的情况，那么怎么替换jdk版本呢？root用户下相对简单，可以直接修改全局变量，但如果是非root用户，或者服务器存在多个用户，需要配置不同的jdk版本，那么就需要修改局部变量了。

1. 全局替换

root用户下，修改全局变量直接参考以下方法：
https://zhidao.baidu.com/question/16233157.html

2. 局部替换

当同一服务器有多个用户，需要不同的JDK版本时，需要修改局部变量（只对当前登录的用户生效）
1）、安装对应的JDK版本；
2）、修改局部变量配置文件：

vim .bash_profile

添加以下内容：

export JAVA_HOME=/usr/java/jdk1.7.0_80
PATH=$PATH:$HOME/bin:${JAVA_HOME}/bin
export PATH

3）、使配置文件生效：

source .bash_profile

4）、查看JDK版本：

java -version

若跟配置的版本一致，则替换正确；
若跟配置的版本不一致，则再次修改局部变量配置文件：

vim .bash_profile

PATH使用绝对路径：

PATH=/usr/lib64/qt-3.3/bin:/usr/local/bin:/bin:/usr/bin:/usr/local/sbin:/usr/sbin:/sbin:/home/jzfw/bin:/usr/java/jdk1.7.0_80/bin:/home/jzfw/bin

source .bash_profile

再次查看JDK版本
