Linux系统xml文件乱码

发表于 2018-05-11 | 更新于 2018-05-14 | 分类于 环境部署 | 评论数：
本文字数： 517 | 阅读时长 ≈ 1 分钟

　　由于终端工具不同，xml文件经过多次编辑之后，文件格式可能发生改变。如果是dos格式，环境启动过程中会报错，此时需要修改文件格式，dos文件表现形式为每一行的行尾会有^M符号。

查看文件格式的两种方法：
1）、file命令

[root@i-C8803F6C test]# file context.xml
context.xml: XML  document text

[root@i-C8803F6C test]# file fdfs_client.conf
fdfs_client.conf: UTF-8 Unicode (with BOM) text

2）、vim打开文件，命令模式下输入： 

:set ff?

返回fileformat=dos或者fileformat=unix

转换文件格式的两种方法：
1）、直接使用转换命令：
dos格式转换为unix格式：dos2unix 文件名
unix格式转换为dos格式：unix2dos 文件名
2）、vim打开文件，命令模式下输入：

:set fileformat=unix
（简写为set ff=unix）
或者:

set fileformat=dos
（简写为set ff=dos）
